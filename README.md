智能会议纪要工具
本项目在 V3.0 的核心架构重构中，深受我老师的开源项目 EthanYoQ/whisper-input 的启发，并在此向其表达诚挚的感谢。

在早期的版本中，本工具采用的是“重型本地部署”架构（内置 FunASR 等离线深度学习模型）。这虽然保障了极致的隐私，但导致了高达 3GB 的安装包体积、极慢的启动速度，并且在交互上需要频繁切换窗口，操作较为繁重。

在学习和借鉴了 whisper-input 倡导的“无感化输入”与“轻量级响应”理念后，我对本项目进行了以下颠覆性重构：

全面云端化与极速瘦身：果断抛弃了笨重的本地 ASR 引擎，全面接入 Gemini 1.5 Pro 的原生音频多模态 API。将“听写”与“总结”双管齐下合二为一，使得软件体积暴降至不足 100MB，实现了真正的“毫秒级秒开”。
引入“全局快捷键”无感交互：借鉴其随时随地调用的设计，新增了 Ctrl+Shift+R 全局录音监听机制。用户即使在全屏处理文档或浏览网页时，也能通过快捷键在后台静默录音，工作流彻底不被打断。
场景化多模板引擎 (Output Styles)：吸收了其“按需输出”的优秀产品定位，打破了以往死板的单一会议总结。我在 UI 左侧引入了下拉模板（包含：详细长篇纪要、精简骨架版、打工人周报版等），让 AI 提炼的结果更贴合真实的职场应用场景。
打通体验的“最后一公里”：补齐了结果输出的交互短板，新增了「一键复制 Markdown」与「导出为 PDF」功能，实现了从语音输入到最终文档落盘的丝滑闭环。
通过这次吸收与改进，本工具正式从一个“笨重的离线听写沙盒”，进化为了一个能够完美融于日常办公流程的“轻量化桌面智能中枢”。
